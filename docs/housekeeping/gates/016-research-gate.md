**Date:** 2026-04-17

**Cycle:** Multi-Issue Batch Cycle (Issues #10-#16)

## Belief-mapping question composed for this gate

At what point in a project's lifecycle does the methodology's own artifact corpus become the kind of "existing implementation" that Issue #16 warns about — and does that change how you think about the relationship between RDD's early phases (establishing the corpus) and its later phases (working within it)?

## User's response

The user distinguished two directionally different evaluation modes: RESEARCH faces outward (expanding the solution space, questioning whether the existing corpus is the right frame) while BUILD faces inward (evaluating whether the existing implementation is doing its job). Connected three existing methodology philosophies — Tidy First, stewardship, simple-vs-easy — as BUILD-phase evaluation lenses, then challenged: "Why do we have the failure modes we do if we have these encoded already?"

This led to the key insight: having the philosophy is not the same as having it fire. The Einstellung finding extends Invariant 8's scope — from "methodology mechanisms need structural triggering" to "practitioner cognitive resources need structural triggering too." The lenses exist; they don't trigger at the right moment because the attentional suppression is pre-reflective.

On premises: the user stated the settled commitment as "we have to encode structurally these mechanisms against attention narrowing" — structural encoding is the first line, cognitive-mechanism design is worth attempting but not the primary defense.

## Pedagogical move selected

Challenge — referenced the essay's treatment of RESEARCH and BUILD interventions as parallel, probed whether the methodology's own corpus becomes the anchoring artifact Issue #16 describes.
